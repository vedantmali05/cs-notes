The basics of Software Engineering and the process of software development in brief.

1.1 Software

Definition - Software

A set of instructions, data or programs used to operate computer and execute specific task is known as a Software

A software is a variable part of a computer and hardware is invariable. Software is anything which is not a hardware. It is developed to achieve goals by Input-Process-Output (End Product). Software plays a dual role:- 1) A Product:- Produce, manage, get, change, display or transmit information. 2) A Vehical for delivering a product:- controls other programs, effects communication, helps build other project, etc.

History of Softwares:- (in short üôÇ) Before 1980s, a software was used to be written on the hardware it ran on. During and after 1980s, the software setup was used to be stored on floppy disks and it was installed in any computer (with all rights.) Later on, CDs and DVDs were introduced for the same purpose. In the present day, the softwares can be directly downloaded using Internet.

Characteristics of Software and Hardware

  • Software is untouchable as it is the set of instructions and hardware is a physical device that can be touched.
  • It doesn't wear out by environmental problems. Dust, temperatures, vibrations, etc affects hardware. :|
  • There are no replacable software spare parts. When hardware fails, its parts can be replaced. :)
  • Software can be custom built but hardware is build assembling existing components.
  • Software is invaluable as it can be installed on any hardware (But technologies like NFT are exceptions) and hardware is useless without a software on it.üòÇ
  • Examples of Software:- Microsoft, Microsoft OS, Internet browsers, video games üéÆ, etc
    Examples of Hardware:- Disks, monitors, keyboards, that mouse or mobile phone in your hand.

  • Generic are open source softwares designed by programmers, Customized one are closed to a organization or firm and are designed by their specification.
  • The developer does all the management of the generic software (like I am maintaining this websiteüòÅ) and Customized softwares are managed by the customer according to his needs.
  • General Purpose Software is tough as it can affect the whole markets requirement which are not known much, so the developer neds to imagine the requirements. Taking surveys and analysing demands may help the programmer. The requirements are already specified by the customer to build a customized software which is why it may feel simple.

    Examples of Customized Software:- Traffic Management System, Process controls systems, project management apps, etc.

Types of Software

  • Application Software:- A software which fulfills a specific need or perform any specific task. So they can be generic or customized. Ex:- File Explorer, Calculator, etc. Programming Software is a sub-category of application software which are used to provide programming tools to developers. Ex:- Microsoft VS Code IDE.
  • System Software:- A software on which computer's hardware works is a System Software. It is a generic software and is a platform for Application Software. Every Operating System like Windows, Linux or macOS are System Softwares. System support software manages hardware efficiently like anivirus programs remove and protect system form viruses. third are System Development Softwares that provide programming development environment to user. For example: Editor is used to create and modify programs, Pre-Processors used to replace some code segments with other codes, Compiler translates high level language to low level and Interpreters do the same task line by line and finally the Loader load program to main memory and execute it.
  • Middleware:- This software lies between System Software and Application Software which then controls the computer devices and its peripherals.
  • Engineering or Scientific Softwares:- These softwares are specially used for drawing, drafting, modeling, heavy calculations, analysis of engineering data, etc. Examples are CAD - Computer-Aided Design, CAM - Computer-Aided Manufacturing and CAE - Computer-Aided Engineering.
  • Embedded Softwares:- The softwares are embedded on a hardware to control its various systems like the finger-print lock üîí of a bicycle. These are embedded in the ROM of the product. The IoT (Internet of Things) is the best example of embedded software. Other examples are washing machine, Amazon's Alexa, Apple's Smart Home Products, etc.
  • Product-line Software:- A product line software is a "set of software system" that "share a common and managed set of features" for "statisfying a special need" of a particular market segment. The features may differ in some ways, example, they may be developed for spcecific customer or market or embedded software.
  • Web Applications:- Accesed via web browsers over Internet or Intranet. These applications are coded as a website in HTML, CSS, JavaScript, ASP, PHP, etc. so that, they can be run without being installed by the client. The information transferred or processed in web apps is public so anyone with internet access and authorization can hunt the data.
    The first generation web apps were static and were usually used to provide information to the client. Meanwhile, "Web 2.0" refers to second generation of web development and has the features of communication, interoperability, user-centered design and collaboration on www (world wide web üåê).
  • Artificial Intelligence Software:- The software which works on AI is a Artificial Intelligence Software. An AI is a computer program which mimics human behavior by learning data patterns and insights. Basically, AI Programs are the algorithms that can create their own algorithms to solve a given complex problem by analysis and interpretation. ü§ñ

Software Engineering as Layered Approach

Softwares are becoming more and more complex and functional due to technological advances. Before, conference calling was not even possible and now hundreds of people connect together to play a game. Hence, now a need of advance from basic programming principle is raised and Software Engineering came to rescue. Well, actually its a part of larger subject called "System Engineering".

Software Engineering Teams are required to study user needs, go designing, implement and test end product.

According to IEE.org, software engineering is "The application of a systematic displined, quantifiable approach to the development, operation and maintenance of software i.e. the application of engineering to software."

üôÇ

Simply

The above statement means - the layered approach of software is - Software Engineering is a systematic approach used by professionals to develop complex software end products within a given time and budget.

Software Engineering includes ‚Ä¢ understanding customer requirements by both the customer and developer ‚Ä¢ use structured methodology to gather customer requirement to its analysis to using Software Requirement Specification (SRS) ‚Ä¢ right estimation of cost, efforts and resources, use of development and testing methods for quality ‚Ä¢ of installation and usage of the software by the customer.

Objectives of softare engineering are ‚Ä¢ Define software development plan ‚Ä¢ Manage development activities ‚Ä¢ Design the proposed product ‚Ä¢ Code or Develop the product ‚Ä¢ Test the program modules ‚Ä¢ Integrate the modules and test the whole system ‚Ä¢ Maintain the product.

4 layers of Software Engineering

  1. Tools:- Contains automated or semi-automated tools offering support to framework and method each Software Engineering project will follow. Ex: Google Developer Tools are the tool used for web development.
  2. Methods:- This contains the technical knowledge and "How-tos..." in order to develop a software.
  3. Process:- Is a framework that must be established for the effective delivery of software.
  4. A Quality Focus:- It is important to test end product according to simplifications like efficiency, usuability, maintenance and reusuability.

1.2 Software Development Framework

üìô

Definition - Framework

The set of activities which are applicable to all software or app projects and domains are Frameworks.

Basically, they are the collection of related actions that produce a major product. And these actions are made by accomplishing work tasks. More simplifying, they are the templates for for every process of Software Development.

Common Phases in all Software Development Process Models:-

  • Communication :- To gather all the requirements a software needs like tools, programmers, designers, etc. heavy communication is carried out with customers and other stakeholders.
  • Planning :- This phase describes the technical tasks to be conducted, risks may raise on the way, resources will be required, end product and work schedule.
  • Modeling:- In this phase the models that let the developers and customers better understand the software requirements and the designs achieving those requirements are created.
  • Construction :- This phase is of code generation, testing and debugging every error in the software.
  • Deployment :- In this phase, the product is presented before customer before launching the final product for feedback and better supply of product. (It is like Beta Mode)
üôÇ

Remember

Remember these phases as the acronym CPMCD - Communication, Planning, Modeling, Construction and Deployment.

All the phases in a software development process together create a Software Development Life Cycleor SDLC.

Umbrella Activities:-

Umbrella Activities are the steps expressed or available in different ways in different software processes followed by a software development teams to maintain progress, quality, changes and risks.

  • Software project tracking and control :- Let the team assess (judge) with project plan and take needed actions to maintain.
  • Risk management :- Assess risks that may affect the end product.
  • Software quality assuarance :- Define and do the activities required to ensure quality of the product.
  • Formal technical reviews :- Resolve or Debug the problems or errors before they spread or affect the next action.
  • Measurement :- Defines and collects process, project and product measures to assist developer team.
  • Software configuration management :- Manage the effect of change throughout the process.
  • Re-usuability management :- Defines criteria for work product reuse.
  • Work product preparation and production :- Encompasses (encircle) the activities required to create work products like models, documentation, etc.

1.3 Software Process Models

A Software Process Model is an abstract representation of a software development process. It describes the "How To Do's" for a software development by specifying development stages and order of a process.

    A software process models defines to establish the end product:-

  • The tasks to be performed
  • Input and Output for each task
  • Pre and Post conditions for each task
  • Flow and sequence of each task
üí°

Goal of a S/W Process Model

The goal of a software process model is to provide guidance for controlling and coordinating the task to achieve the end product and objectives as effectively as possible.

    There are 2 types of Software Process Models:-

  1. Prescriptive Process Models
  2. Specialized Process Models

Prescriptive Process Models

Prescriptive Process Models are the conventional (or typical) process models used when the chosen software engineering approach is described in detail. These models are called "Prescriptive" because they prescribe the set of activities, actions, tasks and modifications for every project. There are 3 Prescriptive Models:-

1. Waterfall Models

Here, each phase must be fully completed before starting next phase. This Model is also known as Linear Sequencial Models aka Classic Life Cycle Models. This model consists of CPMCD - Communication, Planning, Modeling, Construction and Deployment and testing and debugging is done after completing all phases.

    Advantages of Waterfall Models:-

  • Simple to understand and has no uncertain requirements.
  • The phases of the life cycle don't overlap.
  • Works well on the small projects.

    Disadvantages of Waterfall Models:-

  • As testing is carried out the complete life cycle, going back and fixing error is difficult.
  • Risky and Uncertain where requirements are at the riskof changing.
  • Not good for complex and Object Oriented Projects.

2. Incremental Process Models

Here, requirements are divided into various builds making a "multi-waterfall" cycle. These cycles are divided upto smaller and moe managable modules. Each model passes through requirement, design, testing, etc. Then, a working version of software is produced during first module and upcoming modules add more functions and features to previous release. Hence, a working product is available at the early stage.

    Advantages of Incremental Process Models:-

  • A working product is available at the early stage.
  • More flexible - less costly to modify scope and requirement. Hence, it increases risk management.
  • Easy to test and debug as iterations are small.
  • Here, customer can respond to each feed.

    Disadvantages of Incremental Process Models:-

  • requires good planning and design.
  • Total cost is higher than waterfall.

2. Rapid Application Development (RAD) Models

The RAD model is similar to Incremental process model, where components and functions are developed parallely like mini projects. The developments are time boxed, delivered and assembled into working prototype.

    Advantages of RAD Models:-

  • Reduces development time
  • Increases reusuability components
  • Fastens and encourages customer feedback

    Disadvantages of RAD Models:-

  • Depends on stronger team and individual performances.
  • Only the system which can be broke into modules can be made using RAD models.
  • Requireshighly skilled developers or designers.

Specialized Process Models

üìô

Definition - Specialized Process Models

The models made by combining one or more conventional models are Specialized Process Models.

Thus these models has many characteristics of multiple conventional models. It is applicable where a narrowly defined software engineering process approach is chosen.

1.4 Agile Software Development

Also known as Agile meaning flexible (Able to create changes).

üìô

Definition - Agile Software Development

A quick and appropriate response to the change incorporating the changes in software process is done in Agile.

In Agile, the customers and their interactions are focussed more than the tools. The traditional software development models like Waterfall Model take a long time to modify so they cannot be Agile. Thus, usually Agile are based on incremental methods where easy and fast modifications are one of the main advantages.

In Agile, each project is broken up into several iterations in given deadline. Each iteration includes continuous planning, testing, integration. After each iteration, feedback is taken and implemented on upcoming iterations. A working product must be delivered at the end of every iteration.It focusses on keeping code simple, testing often and delivering a function unit as soon as possible.

Advantages of Agile Software Developments:-

  • Predictable cost and time frame. Early and predictable delivery.
  • Improved quality and transperancy.
  • Business engagement or customer satisfaction.
  • Managable, adaptive and reliable end product.

üôÇ Disadvantages of Agile Software Developments:-

  • In Agile, documentation happens throughout a project, and often ‚Äújust in time‚Äù for building the output, not at the beginning. So it is limited in Agile.
  • Fragmented output

Types of Agile Methodologies

1. Adaptive Software Development (ASD)

Adaptive Software Development is a framework typically used in complex softwares. Its life cycle includes.

  • Speculation:- The project is initiated and planned. The planning includes project requirements, user needs and customer mission statement.
  • Collaboration:- Communication and teamwork but emphasizes for creativity.
  • Learning:- Learning helps workers to increase their level of understanding over the project in 3 ways - focussed groups, technical reviews and project postmorten.

2. Extreme Programming (XP)

XP is a disciplined approach focussing on speed and continuous delivery like increased customer involvement, fast feedback loop, continuous planning and testing, etc. High quality product delivery is expected and teams are supported by providing lightweight frameworks to do so.

3. Scrum

In Scrum, the owner creates a product backlog that allows them to work with their team to identify and prioritize system functions. üí° A product backlog is the "To-Do List" or notes having every necessary component or task to deliver a successfull software. The backlog includes details about bug fixes, features list and non-functional requirements. This list is final i.e nothing can be added to it once it is defined. But in some cases, the team can modify the list. Once the backlog is finalized, the team works and delivers one increment in a sprint of usually 30 days. Then after every sprint, the product backlog is re-evaluated, analyzed and prioterized for the next sprint. It is simple and productive to incorporate various practices of other Agile methods in Scrum.

4. Dynamic System Development Method (DSDM)

This is said to be a common industryy framework.

    Defines 8 principles which are super necessary:- (Unaccepting any of the principles will introduce risks)

  1. Collaboration
  2. On-time delivery
  3. Demostrated control
  4. Continuous and clear communication
  5. A constant focus on business need.
  6. Iterative Development
  7. Creation in increments from firm foundations
  8. Refusal to compromise quality.

In DSDM, system requirements are prioritized using MoSCoW - Must have, Should have, Could have, but not critical and Won't have now, but could have later. Here, not every requirement is critical thus they can be removed without impacting the higher priority requirements.

5. Crystal

Crystal is the most lightweight and adaptive methodology which focusses on people and their interactions happening while working on an Agile projec, business critiality and priority of the system under development. It works on the realization that every project possesses unique characteristics (like project priorities, team sizes, system cricalities) that require a slightly tailored üîß set of policies, practices, and processes. Thus, it has different types like Crystal Orange, Crystal Clear and Crystal Yellow. It also emphasizes frequent delivery of a working product with high customer involvement, adaptibility, etc. The priciples of Crystal methodology are - communication, teamwork and simplicity.

1.5 Selection Criteria for Software Process Models

    Selection process parameters play important roles in choosing the best Software Development Life Cycle. They are as follows:-

  1. Requirement Characteristics:- Requirements indicate the complexity of system
    • Reliability of requirements, how often the requirements can change? Types and number of requirements, can they be defined early?
  2. Developer team:-
    • Size, experience of the developers on the similar project, level of understanding of user requirements by developers, developer's environment, domain knowledge of development, experience on the tech to be used, availability of training.
  3. User involvement in the project:-
    • Expertise of user in project, involvement of user in all phases, experience of user in similar project in past.
  4. Project type and associated risks:-
    • Stability of funds, tightness of project schedule, availability of resources, types of project, size of project, expected time frame, complexity of project, level and type of associated risk.
